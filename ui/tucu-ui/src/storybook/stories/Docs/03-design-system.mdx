# Design System

Tucu UI is built on a comprehensive design system that ensures consistency, scalability, and maintainability across all components. Our design system provides a unified visual language that helps create cohesive user experiences.

## Design Principles

### 1. **Consistency**

Every component follows the same design patterns, spacing rules, and visual hierarchy. This creates a predictable and intuitive user experience across different parts of your application.

### 2. **Simplicity**

We believe in clean, uncluttered designs that focus on functionality. Each component is designed to be immediately understandable and easy to use.

### 3. **Flexibility**

While maintaining consistency, our components are designed to be flexible enough to adapt to different use cases and design requirements.

### 4. **Accessibility First**

All components are designed with accessibility in mind, following WCAG guidelines to ensure your applications are usable by everyone.

## Visual Foundation

### **Typography Scale**

Our typography system uses Tailwind CSS utilities with custom extensions:

- **Font Family**: Roboto, monospace (for body text)
- **Font Weights**: Available through Tailwind utilities (font-light, font-normal, font-medium, font-semibold, font-bold)
- **Custom Font Size**: 13px with 18px line height for specific use cases
- **Typography Component**: Available for semantic text rendering (h1-h6, p, blockquote, code, etc.)

### **Spacing System**

Consistent spacing creates visual rhythm and improves scanability using Tailwind's default scale plus custom additions:

```css
/* Standard Tailwind Spacing Scale */
1: 0.25rem     /* 4px */
2: 0.5rem      /* 8px */
4: 1rem        /* 16px */
6: 1.5rem      /* 24px */
8: 2rem        /* 32px */
12: 3rem       /* 48px */
16: 4rem       /* 64px */

/* Custom Extensions */
13: 3.375rem   /* 54px - Custom spacing for specific components */
```

### **Border Radius**

Rounded corners add softness and modernity to the design:

- **None**: 0px - For sharp, technical interfaces
- **Small**: 4px (rounded-sm) - For subtle rounding
- **Default**: 8px (rounded-md, sm:rounded-lg) - Most common use case
- **Large**: 12px (rounded-lg) - For cards and containers
- **Full**: 50% (rounded-full) - For circular elements

## Component Anatomy

### **States**

Every interactive component supports these states:

- **Default**: Normal, uninteracted state
- **Hover**: Mouse hover state with subtle animations
- **Active**: Pressed or selected state
- **Disabled**: Non-interactive state with reduced opacity
- **Loading**: Processing state with loading indicators

### **Variants**

Components come in different variants to suit various contexts:

- **Solid**: Primary actions with filled backgrounds
- **Ghost**: Transparent background for subtle actions
- **Transparent**: Minimal styling with hover effects

### **Sizes**

Consistent sizing across components:

- **Tiny**: `px-2 h-6` / `w-6 h-6` - Minimal space usage
- **Mini**: `px-4 h-8` / `w-8 h-8` - Very compact
- **Small**: `px-7 h-10` / `w-10 h-10` - Compact interfaces
- **Medium**: `px-5 sm:px-8 h-10 sm:h-12` / `h-10 w-10 sm:w-12 sm:h-12` - Standard size
- **Large**: `px-7 sm:px-9 h-11 sm:h-13` / `w-11 h-11 sm:w-13 sm:h-13` - Prominent actions

## Layout Principles

### **Responsive Breakpoints**

Extended breakpoint system for ultra-wide displays:

```css
xs: 500px    /* Extra small */
sm: 640px    /* Small */
md: 768px    /* Medium */
lg: 1024px   /* Large */
xl: 1280px   /* Extra large */
2xl: 1440px  /* 2X large */
3xl: 1780px  /* 3X large */
4xl: 2160px  /* 4K displays */
```

### **Z-Index Scale**

Layered interface elements follow a consistent z-index scale:

- **Base**: z-0 - Normal content
- **Elevated**: z-10 - Tooltips and small overlays
- **Dropdown**: z-20 - Toast notifications
- **Navigation**: z-30 - Sticky headers and navigation
- **Sidebar**: z-40 - Side panels and drawers
- **Modal**: z-50 - Modal dialogs and overlays

- **Large**: `0px 8px 16px rgba(17, 24, 39, 0.1)` - Prominent elements

### **Animations**

Built-in animation system for interactive feedback:

## Theme System

### **Dynamic Theming**

The theme system uses Zustand for state management and supports:

- **Color Presets**: 6 predefined brand colors
- **Mode Switching**: Light/dark mode with automatic detection
- **Layout Options**: Multiple layout configurations
- **Direction Support**: LTR/RTL text direction
- **Persistent Settings**: Local storage integration

### **CSS Custom Properties**

Dynamic brand colors are implemented using CSS custom properties:

```css
:root {
  --color-brand: /* RGB values set dynamically */ ;
}
```

## Best Practices

### **Do's**

- ✅ Use the predefined spacing scale (including custom spacing-13)
- ✅ Follow the established color palette and semantic meanings
- ✅ Utilize the z-index scale for proper layering
- ✅ Use responsive breakpoints including 3xl and 4xl for ultra-wide displays
- ✅ Test components in both light and dark modes
- ✅ Use the shadow system for consistent depth

### **Don'ts**

- ❌ Create custom colors outside the defined palette
- ❌ Use arbitrary spacing values outside the scale
- ❌ Mix different border radius styles inconsistently
- ❌ Ignore the z-index hierarchy
- ❌ Override core design tokens without consideration
- ❌ Use fixed z-index values outside the established scale

## Implementation

The design system is implemented through:

- **Tailwind CSS utilities** with custom extensions for consistent styling
- **CSS custom properties** for dynamic theme switching
- **Zustand state management** for theme persistence
- **TypeScript interfaces** for type-safe component props
- **Component variants** using the cva library pattern
- **Storybook documentation** for visual reference and testing

This systematic approach ensures that every component feels like part of a cohesive whole, making your applications more professional and user-friendly while supporting advanced features like ultra-wide displays and comprehensive theming.
